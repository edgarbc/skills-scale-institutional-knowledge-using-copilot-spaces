# OctoAcme — Role Handoff Checklist Template

Use this checklist when work transitions between functions during planning, execution, release, or escalation.

## How to use this checklist
- Copy the relevant section(s) into an issue, planning note, or release checklist for the current project.
- Mark each item complete before the receiving role accepts ownership.
- Track unresolved items in the decision log with owner and due date.

## Where to store links in the repo
- Add links to supporting artifacts under the backlog item `Related docs/links` field.
- Store shared references in `docs/` (or project-specific folders) and include permalinks in PRs/status updates.

## PdM → UX handoff
- [ ] Problem statement is documented
- [ ] Success metrics and target outcomes are documented
- [ ] Constraints and non-goals are documented
- [ ] Stakeholder or sponsor context is linked
- [ ] PdM and UX confirm design review date

## UX → Dev handoff
- [ ] Wireframes/prototype link is shared
- [ ] Interaction notes and edge cases are documented
- [ ] Acceptance criteria mapping is included
- [ ] Accessibility/usability requirements are noted
- [ ] Dev confirms implementation questions are resolved

## Dev → QA handoff
- [ ] Test plan notes are provided (happy path + edge cases)
- [ ] Feature flags/configuration notes are documented
- [ ] Target environments and test data needs are listed
- [ ] Known limitations/risks are documented
- [ ] QA confirms entry criteria for validation

## Data Analyst ↔ PdM handoff
- [ ] Metric definitions and formulas are agreed
- [ ] Dashboard/report links are shared
- [ ] Baseline values and target thresholds are documented
- [ ] Instrumentation assumptions or gaps are captured
- [ ] PdM/Data Analyst confirm review cadence

## Customer Success → PdM/PM handoff
- [ ] Feedback triage summary is shared (themes + severity)
- [ ] Reproducible examples or customer scenarios are linked
- [ ] Escalation path and owner are confirmed
- [ ] Customer communication needs are identified
- [ ] Next-step decision date is agreed

## PM ↔ Stakeholders handoff
- [ ] Status update cadence is agreed
- [ ] Decision log location is shared
- [ ] Open risks/dependencies and owners are documented
- [ ] Approval checkpoints and sign-off criteria are explicit
- [ ] Escalation contacts and SLAs are confirmed
